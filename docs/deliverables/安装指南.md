# EduAgent 安装部署指南

## 1. 环境要求

| 组件 | 最低版本 | 说明 |
|------|---------|------|
| Docker | 24.0+ | 容器化运行环境 |
| Docker Compose | 2.20+ | 多服务编排 |
| Node.js | 20+ | 前端构建（仅本地开发需要） |
| Python | 3.12+ | 后端运行（仅本地开发需要） |
| 操作系统 | Linux / macOS / Windows (WSL2) | 推荐 Ubuntu 22.04 或 macOS |
| 内存 | ≥8 GB | Milvus 需要较大内存 |
| 磁盘 | ≥20 GB | 数据库和向量索引存储 |

> **注意**：Docker Compose 部署模式下，Node.js 和 Python 无需在宿主机安装，容器镜像已包含。

---

## 2. 快速启动

### 2.1 克隆项目

```bash
git clone https://github.com/xxx/edu-agent-platform.git
cd edu-agent-platform
```

### 2.2 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入必要的 API 密钥和配置项（详见第 3 节）。

### 2.3 启动全部服务

```bash
docker compose up -d
```

首次启动会自动拉取镜像并构建服务，耗时约 5-10 分钟。

### 2.4 查看服务状态

```bash
docker compose ps
```

确认所有 8 个服务状态为 `running`。

## 3. 环境变量说明

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `OPENAI_API_KEY` | 是 | - | OpenAI API 密钥，用于 GPT-5.4 调用 |
| `OPENAI_BASE_URL` | 否 | `https://api.openai.com/v1` | API 代理地址（如使用反代） |
| `DATABASE_URL` | 否 | `postgresql://postgres:postgres@postgres:5432/eduagent` | PostgreSQL 连接字符串 |
| `REDIS_URL` | 否 | `redis://redis:6379/0` | Redis 连接字符串 |
| `NEO4J_URI` | 否 | `bolt://neo4j:7687` | Neo4j 数据库连接地址 |
| `NEO4J_USER` | 否 | `neo4j` | Neo4j 用户名 |
| `NEO4J_PASSWORD` | 是 | - | Neo4j 密码（首次需设置） |
| `MILVUS_HOST` | 否 | `milvus` | Milvus 向量数据库地址 |
| `MILVUS_PORT` | 否 | `19530` | Milvus 端口 |
| `JWT_SECRET` | 是 | - | JWT 签名密钥（随机字符串） |
| `LTI_PLATFORM_URL` | 否 | - | 超星平台 LTI 1.3 配置 URL |
| `LTI_CLIENT_ID` | 否 | - | LTI 1.3 Client ID |
| `LTI_KEY` | 否 | - | LTI 1.3 平台密钥 |
| `DINGTALK_APP_KEY` | 否 | - | 钉钉应用 AppKey |
| `DINGTALK_APP_SECRET` | 否 | - | 钉钉应用 AppSecret |
| `DINGTALK_ROBOT_CODE` | 否 | - | 钉钉机器人 Code |
| `EMBEDDING_MODEL` | 否 | `text-embedding-3-large` | 向量化模型名称 |
| `EMBEDDING_DIM` | 否 | `3072` | 向量维度 |
| `NEXT_PUBLIC_API_URL` | 否 | `http://localhost:8000` | 前端调用后端 API 地址 |
| `NEXT_PUBLIC_COPILOTKIT_URL` | 否 | `/api/copilotkit` | CopilotKit Runtime 端点 |

## 4. 初始化数据

服务启动后，执行种子数据初始化：

```bash
# 初始化数据库表结构
docker compose exec backend python -m alembic upgrade head

# 导入种子数据（演示课程、知识点、题库、测试用户）
docker compose exec backend python -m data.seed
```

种子数据包含：
- **2 门演示课程**：高等数学、大学物理
- **50+ 知识点**：含跨课程关联关系（CROSS_COURSE）
- **100+ 练习题**：按知识点分类
- **测试用户**：教师账号 `teacher@demo.com` / 学生账号 `student@demo.com`（密码均为 `demo123`）

### 4.1 初始化知识图谱

```bash
# 将知识点和关系导入 Neo4j
docker compose exec backend python -m data.seed_neo4j
```

### 4.2 构建向量索引

```bash
# 将课程文档解析、分片并索引到 Milvus
docker compose exec backend python -m data.seed_milvus
```

## 6. 本地开发

如需在宿主机直接运行前后端进行开发调试：

### 6.1 启动基础设施

```bash
# 仅启动数据库等基础服务
docker compose up -d postgres neo4j milvus redis
```

### 6.2 启动后端

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

### 6.3 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端开发服务器默认运行在 http://localhost:3001。

## 7. 常见问题

### Q1: Milvus 启动失败 / 内存不足

Milvus standalone 模式需要至少 4GB 内存。检查 Docker 内存限制：

```bash
docker stats --no-stream milvus
```

若内存不足，在 `docker-compose.yml` 中调整 `deploy.resources.limits.memory`，或增大 Docker Desktop 的内存配额。

### Q2: Neo4j 认证失败

首次启动 Neo4j 需要设置密码。确保 `.env` 中的 `NEO4J_PASSWORD` 已设置，且与 `docker-compose.yml` 中的环境变量一致：

```yaml
neo4j:
  environment:
    NEO4J_AUTH: neo4j/${NEO4J_PASSWORD}
```

### Q3: LTI 1.3 对接超星配置

1. 在超星管理后台注册 LTI 1.3 外部工具
2. 获取 Platform URL、Client ID 和密钥
3. 填入 `.env` 对应变量
4. 重启 lti-provider 服务：`docker compose restart lti-provider`

### Q4: 前端无法连接后端 API

检查 Nginx 反向代理配置是否正确转发 `/api` 请求到 backend 服务：

```bash
docker compose logs nginx | tail -20
```

确认 `nginx.conf` 中的 upstream 配置与服务名一致。

### Q5: 向量索引构建缓慢

大量文档索引时，可调整批处理参数：

```bash
docker compose exec backend python -m data.seed_milvus --batch-size 100
```

### Q6: GPT API 调用超时

若使用代理访问 OpenAI API，确保 `OPENAI_BASE_URL` 配置正确。可通过以下命令测试连通性：

```bash
docker compose exec backend python -c "
from openai import OpenAI
client = OpenAI()
print(client.models.list().data[0].id)
"
```

### Q7: Docker Compose 版本不兼容

确保使用 Docker Compose V2（`docker compose` 命令，非 `docker-compose`）。检查版本：

```bash
docker compose version
# 需要 v2.20.0 或更高
```

## 8. 生产部署建议

| 项目 | 建议 |
|------|------|
| SSL 证书 | 使用 Let's Encrypt + Certbot 自动续期 |
| 域名 | 配置 Nginx server_name 和 SSL |
| 数据备份 | PostgreSQL pg_dump + Neo4j dump 定时备份 |
| 日志 | 配置 Docker 日志驱动，接入 ELK 或 Loki |
| 监控 | 添加 Prometheus + Grafana 监控 |
| 安全 | 修改所有默认密码，限制端口暴露 |
